Tutorial 1: the EULA

A signatory fills in a Google Form and get back an agreement to sign. They don't even have to know Legalese exists.

  1. In this tutorial, you see how Legalese might appear to a third-party signatory.
  2. Legalese.io takes the role of a Company, providing you an End-User License Agreement.
  3. You take the role of an end-user signing the License Agreement.
  4. In future, you will take the role of the Company generating agreements for others.
  5. Legalese.io invites you to fill in a simple form with your name and email address.
  6. A few minutes later, check your mail. You should see a PDF agreement there. If you don't receive the EULA, that's because there is a bug in the software that does OAuth integration with EchoSign. Sorry, this is still alpha software. We're working on it.
  7. To complete the tutorial, sign the EULA.

After running this tutorial, you will have: seen what Legalese's Particulars form looks like to a third-party signatory, seen the output PDF, and signed it.

Tutorial 2: Hello, World!

You set up a Yellow Spreadsheet for the first time. You are now a power user!

  1. Copy our template to your Google Drive account.
  2. (Say yes to "Make a copy?")
  3. You should be in the Hello World tab. Some of the cells are bright yellow. We chose the same shade of yellow that you've seen in Word docs where you're expected to fill in the blanks.
  4. Go ahead and fill in your particulars in the blank Row 14. If you leave it blank, we'll make something up for you. Change the email address for the Acme Example company to be one of your email addresses.
  5. What about the cells that aren't in yellow? You can look, but don't touch. Yet.
  6. Create an agreement! Click Add-ons / Legalese / Generate PDFs. Approve the authorization request.
  7. Cross your fingers. Because sometimes it doesn't work. Usually it's Google's fault. But sometimes it's our fault. We're in beta. :-/
  8. If it didn't work, try again. If it still doesn't work, get support.
  9. Enough pessimism! Let's assume it worked. A link will appear in cell E6 of the spreadsheet. It leads to an output folder in Google Drive.
  10. In that folder, you should see an XML file. Yay! You are not expected to understand the XML. That's the job of the Legalese Robot.
  11. Wait three minutes for the Legalese Robot to turn the XML into a PDF.
  12. (While you wait, you might check out the README file.)
  13. If, after three minutes, a PDF doesn't appear, just reload the web page.

After running this tutorial, you will have: gotten comfortable with Legalese's Yellow Spreadsheets, created a PDF of your own, read the README, and glanced at the XML source files.

I noticed the output folder was shared with robot@legalese.io. What if I have confidential information? Very perceptive of you! The Legalese Robot operates without any human involvement, turning XML into PDF tirelessly, day and night. In the normal course of operation, no human views your agreements. If debugging or troubleshooting is needed, only with your permission will a human view your XML or PDF files. (Errors are logged to a separate file which help us debug without ever viewing your agreements.)

Are the yellow spreadsheets shared? No, the yellow spreadsheets are not shared with the Legalese Robot. The robot only has access to your XML, which it needs to generate the PDFs.

Tutorial 3: Send for Signature

You, a power user, inject the Hello World PDF into an e-signature service which emails it to you for signing.

  1. Open the Google Drive folder that contains the Hello World PDF.
    • The folder is named My Legalese Tutorial Hello World.
    • (If you don't know how to find the folder, begin by going back to the yellow spreadsheet.)
    • (If you don't know how to find the yellow spreadsheet, search your Google Drive for My Legalese Tutorial.)
    • (If you don't want to search, start at My Drive and open Legalese Root. Everything that Legalese does is in there. We created this folder for you automatically.)
  2. Select the Hello World PDF.
  3. Right-click the PDF (or hit a if you're a keyboard geek).
  4. Open With: E-Sign with EchoSign.
  5. If you don't see the EchoSign option, click Connect More Apps to add it.
  6. Enter the To: and CC: addresses based on what's shown in the README file. The order matters.
  7. If you're using EchoSign, you'll notice that the signature blocks are already in the right place.
  8. Go ahead and send for signature!

After running this tutorial, you will have: set up an e-signature back-end and sent a PDF for signature.

When the signature service asks me for the signers' email addresses, what should I say? Open up the README file and enter the email addresses shown in the To: line.

What if I prefer DocuSign / HelloSign / some other e-signature backend? That's fine; you can certainly do that. The advantage of EchoSign is that the PDFs contain embedded Signature Blocks which EchoSign knows how to read. If you use some other signature service, you will need to manually place the signature blocks.
